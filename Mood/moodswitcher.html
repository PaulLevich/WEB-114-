1. Why is the JavaScript textContent property safer than innerHTML?

2. If use an HTML text box on a page and it has an attribute (for example id="text") and a corresponding DOM property--for example, const box = document.getElementById("text"). How do the two elements relate?

id provides a unique "name tag" while document.getElementById() uses that "name tag" to locate and return a reference.

3. What would happen to the text box with id="text" if it added this line to the code above: text.textContent = "Hello!"

The existing content within the text box would be replaced and the text box would then display "Hello!".
